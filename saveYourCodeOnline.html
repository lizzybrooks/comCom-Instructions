Creating a GitHub Repository for Node Projects

Git is a version control system that allows you to save changes to your code without losing previous drafts. Github is an online storage place for code that uses the git system. Using this system, you can work on a project and keep track of your changes. It takes a minute to set up, but it's worth it.

Initialize your project

Open Terminal (mac) or the command prompt (windows). Navigate to your project's root directory (if you open your terminal through VSCode, you'll already be in your project's root directory). Type the following command (or copy and paste it) to initialize the git process inside this project 

git init

Create .gitignore file

A .gitignore file is literally a file that tells the git process which files in your project it should ignore, or not upload to Github. We want to ignore the node files because they're huge and there are too many of them to store online, and also because it's redundant to store them online. Anyone who wants to open the project will use your package.json file to install these additional libraries (called dependencies) on their own computer.


In VSCode, create a new file, call it .gitignore and put the below text into it. Then save it. 

node_modules/

Add your project files to the repository

Run this series of commands in your terminal window to add and commit your files to Git:

git add .
git commit -m "Initial commit"

Create a repository on GitHub

Create a new repository on GitHub via the website. Go to Github.com, log in and go to the + --> new repository place menu on the upper right. Name your repository whatever you want. Don't do any of the setup on the website. You are doing this process on your local computer, using terminal or command prompt.

Link your local repository to GitHub

Run the following code to link your local repository with the GitHub repository: (replace the dummy text with your actual URL for your repository, copied and pasted from the url bar at the top of the browser).

git remote add origin <REPOSITORY_URL>
git branch -M main
git push -u origin main

Run the following script to push your local commits to the GitHub repository. This means that you will push or upload your files into the online storage space, GitHub. 

git push -u origin main

Click in the code tab in your repository to make sure the files are there.
